O protocolo MESI garante que nunca haverá dois núcleos com uma cache line no estado M simultaneamente para o mesmo endereço. Essa invariante é a "coerência" — qualquer leitura sempre retorna o valor mais recente, independente de qual núcleo o escreveu.
🟡 M — Modified
A cache line foi modificada e está diferente da RAM. Só este núcleo tem o valor atual — a RAM está desatualizada (stale). Antes de qualquer outro núcleo ler este endereço, este núcleo deve fazer writeback para a RAM (ou fornecer o dado diretamente via snooping). Só um núcleo pode estar em M para um dado endereço.
🟢 E — Exclusive
A cache line está igual à RAM, e só este núcleo a tem. Nenhum outro núcleo tem cópia. Como é exclusiva, este núcleo pode escrever nela e ir direto para estado M — sem precisar avisar ninguém no barramento. É o estado "fast path" para escrita.
🔵 S — Shared
A cache line está igual à RAM e outros núcleos também podem ter cópias. Múltiplos núcleos podem estar em S para o mesmo endereço. Para escrever, deve-se primeiro invalidar as cópias dos outros núcleos (BusUpgr ou BusRdX) e transitar para M. Leituras são livres e não geram tráfego no barramento.
🔴 I — Invalid
A cache line não tem dado válido. Pode ser porque nunca foi carregada, ou porque outro núcleo a invalidou. Qualquer acesso (leitura ou escrita) causa cache miss e aciona o protocolo para buscar o dado — da RAM ou do núcleo que tem a versão atual (snooping).

MESI é o protocolo base. Variantes adicionam estados: MOESI adiciona O (Owned — modificado mas compartilhado), MESIF adiciona F (Forward — quem fornece o dado a outros). Processadores AMD usam MOESI, Intel usa MESIF. O L3 muitas vezes atua como "last-level snoop filter" — intercepta requests antes de chegarem à RAM.
